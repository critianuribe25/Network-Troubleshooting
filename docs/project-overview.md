This project simulates real-world network troubleshooting using the Windows Command Prompt.  
It focuses on diagnosing connectivity and DNS issues using standard networking tools.

## Key Objectives
- Review local IP configuration
- Test connectivity to external IP addresses
- Verify DNS resolution
- Trace the network route to a remote host
- Practice a structured troubleshooting approach

## Why This Matters
Cloud and IT support roles require strong troubleshooting skills.  
Knowing how to quickly identify whether a problem is local, network-related, or DNS-related is critical for resolving user issues efficiently.
